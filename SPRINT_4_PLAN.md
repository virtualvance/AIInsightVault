# Sprint 4: Agnostic Persistence Pipeline
**Goal:** Transition from Logcat logging to a provider-agnostic Database storage system.

## 1. Data Schema (Agnostic Design)
- **Primary Key:** `id` (String) - A generated unique ID to ensure independence from AI provider URLs.
- **Entity:** `InsightEntity`.
- **Fields:**
  - `title`: User-defined or AI-suggested title for the conversation.
  - `source`: The AI provider used (e.g., "Gemini").
  - `rawContent`: The full captured conversation transcript.
  - `summary`: High-level summary generated by the local AI processor.
  - `tags`: AI-suggested categorization tags for searching/filtering.
  - `timestamp`: Long (System time of the last capture/update).

## 2. Persistence Strategy
- **Upsert Logic:** `VaultDao` uses `OnConflictStrategy.REPLACE`. If a match is found, it updates the transcript.
- **Migration:** Use `fallbackToDestructiveMigration()` during development to cleanly wipe data when schema changes occur.

## 3. Local AI Integration Pipeline (Phased)
- **Phase 1 (Current):** Implement a `MockSummarizationService` using Kotlin Coroutines (`delay`) to test UI and Database wiring without risking heavy Gradle dependencies.
- **Phase 2 (Future):** Replace the Mock with LiteRT-LM (Gemma 4 model) for true on-device, offline summarization.

## 4. Execution Workflow
1. **Capture:** Execute the bridge script to extract the transcript.
2. **Process:** Pass text to the Summarization Service (Mock first, LiteRT-LM later).
3. **Confirm:** Present the Sprint 4 "Save Vault" UI showing the summary/tags for meaningful human control.
4. **Store:** Save the structured `InsightEntity` object to the Room Database.

## 5. Agent Interaction Rules (Planning Mode)
- **Rule 1:** Always propose a step-by-step plan and wait for human approval before modifying code.
- **Rule 2:** Do not add new Gradle dependencies unless explicitly instructed.
