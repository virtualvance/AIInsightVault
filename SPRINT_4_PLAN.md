# Sprint 4: Agnostic Persistence Pipeline
**Goal:** Transition from Logcat logging to a provider-agnostic Database storage system.

## 1. Data Schema (Agnostic Design)
- **Primary Key:** `id` (String) - A generated UUID to ensure independence from AI provider URLs.
- **Fields:**
    - `title`: User-defined or AI-suggested title for the conversation.
    - `source`: The AI provider used (e.g., "Gemini").
    - `rawContent`: The full captured conversation transcript.
    - `summary`: High-level summary generated by the local AI processor.
    - `tags`: AI-suggested categorizatme of the last capture/update).

## 2. Persistence Strategy
- **Upsert Logic:** The app will check for existing titles; if a match is found, it updates the transcript. If not, it creates a new entry with a unique UUID.
- **Migration:** Use `fallbackToDestructiveMigration()` during development to cleanly wipe current data and apply this new schema.

## 3. Local AI Integration (LiteRT-LM)
- **Engine:** LiteRT-LM (The highion tags for searching/filtering.
    - `timestamp`: Long (System ti-performance successor to MediaPipe LLM Inference).
- **Model:** Gemma 4 (Optimized for on-device reasoning and summarization).
- **Workflow:** Captured text is passed to the LiteRT-LM engine to generate a summary and tags before being committed to the database.

## 4. Execution Workflow
1. **Capture:** Execute the bridge script to extract the transcript.
2. **Process:** LiteRT-LM summarizes the text on-device (Offline-first).
3. **Confirm:** Present a "Save Insight" UI showing the summary and tags.
4. **Store:** Save the structured `Insight` object to the Room Database.
